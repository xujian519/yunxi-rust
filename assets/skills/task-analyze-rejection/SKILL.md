# 任务2_2：驳回理由深度分析

> **场景**: 答复审查意见
> **步骤**: 第2步 - 驳回理由深度分析
> **版本**: v1.0
> **创建时间**: 2025-12-26
> **设计者**: YunXi Agent v1.0

---

## 任务概述

**任务名称**: 驳回理由深度分析

**任务目标**:
- 针对每个驳回理由进行法律分析
- 从向量库检索相关法条和案例
- 评估驳回理由的成立可能性
- 为答复策略制定提供依据

**输入材料**:
- TASK_2_1的驳回理由清单
- 引用的对比文件
- 本申请的申请文件

**输出成果**:
- 每个驳回理由的详细分析报告
- 成立可能性评估
- 相关法律依据和参考案例

---

## 执行流程

### 针对每个驳回理由的分析框架

```
对于驳回理由X（例如：权利要求1的创造性问题）

├─ 第一步：法律检索
│   └─ 从向量库检索：
│       ├─ patent_rules_complete: 相关法条和审查指南
│       ├─ patent_decisions: 类似案例
│       └─ patent_guidelines: 审查指南相关章节
│
├─ 第二步：新颖性分析（如适用）
│   ├─ 确定最接近现有技术
│   ├─ 识别区别特征
│   └─ 判断是否被单独公开
│
├─ 第三步：创造性分析（如适用）
│   ├─ 第一步：确定最接近的现有技术
│   ├─ 第二步：确定区别特征
│   └─ 第三步：判断是否有技术启示
│
├─ 第四步：其他问题分析
│   ├─ 公开不充分（A26.3）
│   ├─ 不清楚（A26.4）
│   └─ 修改超范围（A33）
│
└─ 第五步：成立可能性评估
    ├─ 评估驳回理由成立的概率
    ├─ 评估成功答复的可能性
    └─ 给出初步建议
```
