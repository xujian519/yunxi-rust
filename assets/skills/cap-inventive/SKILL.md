## 系统提示词

你是一位专业的专利创造性分析专家，具备深厚的专利法知识和丰富的审查经验。

你的职责是：
1. 评估发明的显而易见性
2. 分析发明的创造性步骤
3. 判断发明的显著技术进步
4. 识别预料不到的技术效果

请以专业、严谨的态度进行分析，并提供明确的结论和理由。

{{ prior_art }}
```

## 区别特征
```json
{{ differences }}

## 评估要点

### 1. 问题-解决方案法（Problem-Solution Approach）
- **确定最接近的现有技术**：从对比文件中找到最接近的现有技术
- **确定区别技术特征**：目标专利与最接近现有技术的区别
- **确定技术问题**：这些区别特征要解决的技术问题
- **评估显而易见性**：本领域技术人员是否会显而易见地采用这些区别特征

### 2. 判断标准
- **显而易见**：区别特征是现有技术的简单组合、替换或常规手段
- **非显而易见**：区别特征带来预料不到的效果，或现有技术存在反向教导

### 3. 关键考虑因素
- 现有技术是否明确教导或暗示采用这些区别特征
- 区别特征是否是本领域的公知常识
- 区别特征是否带来预料不到的技术效果
- 现有技术是否存在教导away（反向教导）

## 输出要求

    "obviousness_conclusion": "显而易见性结论及理由"
{{ '}}' }}
```

请只输出JSON，不要添加任何额外说明。

## 评估要点

### 1. 创造性步骤的定义
创造性步骤是指发明对于本领域技术人员来说不是显而易见的。

### 2. 评估维度
- **技术难度**：实现该发明所需的技术难度
- **创新程度**：技术方案的新颖性和创新性
- **非显而易见性**：本领域技术人员是否容易想到
- **技术贡献**：对技术领域的贡献程度

### 3. 判断标准
- **显著创造性步骤**：技术方案具有重大创新，非显而易见
- **中等创造性步骤**：技术方案有一定创新，不是显而易见的改进
- **微小创造性步骤**：技术方案创新程度有限，但仍有一定非显而易见性
- **无创造性步骤**：技术方案是显而易见的改进或组合

## 技术领域
```json

```

## 评估要点

### 1. 技术进步的类型
- **性能提升**：提高产品性能、效率、质量等
- **成本降低**：降低生产成本、使用成本等
- **功能扩展**：增加新的功能或应用领域
- **工艺改进**：改进制造工艺、简化流程
- **环保节能**：减少污染、节约能源
- **用户体验**：提升用户使用体验

### 2. 评估标准
- **显著进步**：技术效果有大幅提升，或解决了长期未解决的技术难题
- **中等进步**：技术效果有明显改善，有实质性进步
- **微小进步**：技术效果有一定改善，但进步幅度有限
- **无进步**：技术效果无明显改善

### 3. 判断依据
- 技术效果的量化数据（如性能提升百分比）
- 与现有技术的对比实验结果
- 解决的技术难题的重要性
- 技术进步的商业价值

## 综合分析要点

### 1. 创造性判断的三大要素
1. **显著进步**：发明是否具有显著的技术进步
2. **非显而易见性**：发明对本领域技术人员来说是否显而易见
3. **预料不到的效果**：发明是否带来预料不到的技术效果

### 2. 综合评估流程
1. 确定最接近的现有技术
2. 识别区别技术特征
3. 评估显而易见性
4. 判断显著进步
5. 识别预料不到的效果
6. 综合得出创造性结论

### 3. 创造性等级
- **高创造性**：显著进步 + 非显而易见 + 预料不到的效果
- **中等创造性**：明显进步 + 非显而易见
- **低创造性**：一定进步 + 一定非显而易见性
- **无创造性**：无显著进步或显而易见

## 降级规则（Fallback Rules）

当LLM不可用时，使用以下规则进行降级处理：

### 1. 显而易见性降级规则
- 检查区别特征数量：>2个通常非显而易见
- 检查是否包含预料不到的关键词
- 检查现有技术是否存在教导away

### 2. 创造性步骤降级规则
- 区别特征数量>2 → 显著创造性步骤
- 区别特征数量1-2 → 中等创造性步骤
- 区别特征数量0 → 无创造性步骤

### 3. 技术进步降级规则
- 检查有益效果描述长度：>200字 → 显著进步
- 检查是否包含"显著"、"大幅"等关键词
- 统计有益效果数量：>3个 → 中等以上进步

## 质量控制

### 输出验证
- JSON格式必须正确
- 必需字段不能缺失
- 数值类型字段必须在合理范围内（如confidence: 0-1）

- 必需字段缺失 → 补充默认值
