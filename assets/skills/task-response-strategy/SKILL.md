# 任务2_3：答复策略制定

> **场景**: 答复审查意见
> **步骤**: 第3步 - 答复策略制定
> **版本**: v1.0
> **创建时间**: 2025-12-26
> **设计者**: YunXi Agent v1.0

---

## 任务概述

**任务名称**: 答复策略制定

**任务目标**:
- 基于驳回理由分析结果，制定具体答复策略
- 为每个驳回理由确定答复方向和策略组合
- 评估各策略的成功率和风险
- 为答复文本撰写提供明确指导

**输入材料**:
- TASK_2_1的驳回理由清单
- TASK_2_2的深度分析报告
- 本申请的申请文件
- 对比文件副本

**输出成果**:
- 每个驳回理由的答复策略
- 策略实施的具体步骤
- 成功率评估和风险提示
- 答复时间规划

### 第三步：详细策略制定

#### 策略A：完全反驳策略

**适用场景**:
- 审查员观点明显错误
- 对比文件未被正确理解
- 法律适用有误

**策略模板**:

```markdown
## 驳回理由X的答复策略：完全反驳

### 答复方向
完全反驳审查员观点，认为本申请具备可专利性

### 答略组合
1. **澄清说明** (40%)
   - 澄清权利要求术语的真实含义
   - 说明本申请技术方案的真实内容
   - 指出审查员理解上的偏差

2. **对比分析** (40%)
   - 详细对比本申请与对比文件
   - 强调区别特征
   - 说明区别特征带来的技术效果

3. **法律依据** (20%)
   - 引用相关法条和审查指南
   - 引用类似案例支持观点

### 实施步骤
1. 准备对比分析表
2. 撰写技术对比说明
3. 收集支持性法条和案例
4. 撰写答复正文

### 成功率评估
- 成功可能性: 70%
- 主要风险: 审查员可能坚持原观点
- 风险缓解: 准备充分的对比分析和技术论证

### 预计答复时间
- 准备时间: 2-3天
- 撰写时间: 1-2天
- 总计: 3-5天
```

#### 策略B：部分反驳 + 修改策略

**适用场景**:
- 审查员观点部分正确
- 可以通过修改克服驳回
- 修改后仍能保持核心保护范围

**🤝 人机交互点2：修改方案选择**

```
【云熙】用户，针对驳回理由2（创造性问题），我为您准备了多个修改方案：

📊 修改方案对比：

┌─────┬──────────────────┬──────────┬────────────┬────────────┐
│方案 │   修改内容        │ 保护范围 │ 成功概率   │  风险评估   │
├─────┼──────────────────┼──────────┼────────────┼────────────┤
│方案A│加入特征A到权利要求│ 缩小5%   │  85%       │ 低风险     │
│     │1，保持从属权利要  │          │            │            │
│     │求2-5不变          │          │            │            │
├─────┼──────────────────┼──────────┼────────────┼────────────┤
│方案B│加入特征A+B到权利  │ 缩小10%  │  95%       │ 极低风险   │
│     │要求1，调整从属权  │          │            │            │
│     │利要求             │          │            │            │
├─────┼──────────────────┼──────────┼────────────┼────────────┤
│方案C│删除权利要求1，以  │ 缩小15%  │  99%       │ 几乎无风险 │
│     │权利要求2为基础重  │          │            │            │
│     │新撰写             │          │            │            │
└─────┴──────────────────┴──────────┴────────────┴────────────┘

📋 详细分析：

**方案A - 最小改动方案**
修改内容:
- 在权利要求1中加入特征A
- 保持从属权利要求2-5不变

保护范围影响:
- 缩小约5%，仍保持较宽保护范围
- 核心技术方案得到保护

成功原因:
- 特征A是说明书中明确记载的
- 加入特征A足以区别D1
- 审查员很可能接受此修改

**方案B - 平衡方案（推荐）**
修改内容:
- 在权利要求1中加入特征A+B
- 调整从属权利要求层次

保护范围影响:
- 缩小约10%，保护范围适中
- 核心创新点得到充分保护

成功原因:
- 双重区别特征，论证更有力
- 仍然保持较合理的保护范围
- 成功率极高

**方案C - 保守方案**
修改内容:
- 删除权利要求1
- 以权利要求2为基础重新撰写

保护范围影响:
- 缩小约15%，保护范围较窄
- 但核心创新点仍受保护

成功原因:
- 最保守的修改方式
- 几乎可以保证克服驳回
- 但保护范围损失较大

【您倾向于哪个方案？】
A. 方案A - 最小改动
B. 方案B - 平衡方案（推荐）
C. 方案C - 保守方案
D. 我有其他想法
```

#### 策略C：接受 + 修改策略

**适用场景**:
- 审查员观点基本正确
- 驳回理由确实成立
- 必须通过修改才能授权

**策略模板**:

```markdown
## 驳回理由X的答复策略：接受 + 修改

### 答复方向
接受审查员观点，通过修改申请文件克服驳回

### 策略组合
1. **承认事实** (20%)
   - 礼貌地表示理解审查员观点
   - 不激烈对抗，保持良好沟通

2. **说明修改依据** (50%)
   - 详细说明修改内容
   - 说明修改来自原始公开
   - 符合A33修改要求

3. **强调修改后优点** (30%)
   - 说明修改后的技术方案
   - 强调仍然具备的创造性
   - 说明保护范围仍然合理

### 实施步骤
1. 确定修改内容
2. 准备修改对照页
3. 撰写答复正文
4. 准备修改后的申请文件

### 成功率评估
- 成功可能性: 95%+
- 主要风险: 修改可能被认定超范围
- 风险缓解: 严格依据原始公开内容修改

### 预计答复时间
- 准备时间: 1-2天
- 撰写时间: 1天
- 总计: 2-3天
```

```
- 优点：可以在答复中补充说明
- 缺点：需要提供证据证明是公知常识

- 优点：成功率最高

3. 策略A仅当我们有十足把握时使用

【您希望如何处理这个问题？】
A. 先让我评估公开不充分是否真的成立
B. 直接选择策略B

D. 我有其他想法
```

```markdown
- 用词要精准，避免再次产生歧义
- 参考说明书的定义
- 保持保护范围的合理性
```

```markdown
## 修改超范围问题的答复策略
```
