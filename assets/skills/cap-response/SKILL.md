## 能力描述

你能够针对审查意见撰写答复策略和答复文本，分析驳回理由，构建反驳论点，必要时建议权利要求修改。

### 步骤1: 分析审查意见的驳回理由

```python
    """分析审查意见"""

        # 驳回依据
```

```python
def retrieve_response_support(analysis):
    """检索答复支撑材料"""

    # 检索相关法条
    laws = qdrant_search("patent_rules_complete", analysis["legal_basis"], top_k=10)
```

### 步骤3: 构建答复论点

```python
def build_response_arguments(analysis, laws, cases):
    """构建答复论点"""

    arguments = []

    for reason in analysis["rejection_reasons"]:
        # 反驳策略选择
        if reason["type"] == "事实认定错误":
            arguments.append(build_fact_correction_argument(reason, laws, cases))
        elif reason["type"] == "法律适用错误":
            arguments.append(build_legal_correction_argument(reason, laws, cases))
        elif reason["type"] == "技术启示判断":
            arguments.append(build_teaching_argument(reason, laws, cases))
        # ... 其他类型

    return arguments
```

### 步骤4: 必要时建议权利要求修改

```python
def suggest_claim_modification(analysis, arguments):
    """建议权利要求修改"""

    modifications = []

    # 基于审查意见建议修改
    for reason in analysis["rejection_reasons"]:
        if reason["suggestion_modification"]:
            mod = {
                "original_claim": reason["target_claim"],
```

## 答复策略建议

### 策略1: 全面反驳

**适用场景**: 审查员观点明显错误
**答复重点**: 逐一反驳所有驳回理由
**修改建议**: 不修改或最小修改

### 策略2: 部分承认+部分反驳

**适用场景**: 部分驳回理由合理
**答复重点**: 承认合理部分，反驳不合理部分
**修改建议**: 针对合理部分进行修改

### 策略3: 修改+反驳

**适用场景**: 驳回理由有一定道理
**答复重点**: 主要通过修改克服缺陷
**修改建议**: 较大修改，进一步限定
